Архитектурные подходы

Подход А: 1 VPS на 1 сотрудника

Полная изоляция
Каждый человек работает на своей машине. Если скомпрометирован один — остальные не пострадали. Собственный IP, своя файловая система, независимый контроль.
Плюсы
Изоляция, разные IP, легко выключить при увольнении, автоматизация через API, RBAC по-машинно.
Минусы
Нужен скрипт провизии (Ansible/Terraform), больше машин для мониторинга, чуть дороже чем терминальный сервер.
✓ Это то, что вы описали. Рекомендую именно этот подход.

Подход Б: Terminal Server (VDI)

1–2 мощных сервера = рабочие столы для всех
Hetzner AX102 (12 ядер, 128 GB RAM) ~€100/мес. На 2 сервера (~€200/мес) можно посадить всех 40 человек через RDP/NoMachine.
Плюсы
Очень дёшево (~€200/мес на всех), легче обновлять систему, централизованный бэкап.
Минусы
Все на одной машине — один взлом = всё сломалось. Нет изоляции, один IP у всех, нагрузка делится.
⚠ Дёшево, но слабо с точки зрения безопасности iGaming-модели.

Безопасность как у iGaming / казино компаний

Почему казики — эталон корпоративной безопасности

iGaming-компании работают в высококонкурентной среде с большими деньгами, часто в нескольких юрисдикциях. Они становятся мишенью для атак конкурентов, госструктур и хакеров. Поэтому их стандарты безопасности — одни из самых жёстких в корпоративном мире.

1 сотрудник = 1 машина Никакого BYOD. Только корпоративный VPS/ноутбук. При увольнении — немедленный reset. Используют MDM (Intune / Jamf) для контроля политик.
Zero Trust Network Ни одному устройству не доверяют по умолчанию. Весь доступ — через Cloudflare Access или Tailscale, с подтверждением личности + устройства.
VPN с kill switch на каждой машине WireGuard / OpenVPN на каждом VPS. Если VPN отвалился — интернет блокируется автоматически (iptables / ufw). Никакого трафика в открытую.
MFA на всё без исключений Google Authenticator / Authy для повседневного доступа. YubiKey (hardware token) для входа в критичные системы: финансы, продакшн, БД.
Мониторинг и SIEM auditd / Wazuh / Graylog собирают логи с каждой машины. Алерты на: root-вход, sudo-команды, нетипичный трафик, новые процессы в автозагрузке.
DLP — запрет утечек данных Data Loss Prevention: блокировка загрузки файлов на личные облака, запрет подключения USB (если физическое устройство), мониторинг буфера обмена.
Шифрование диска и каналов LUKS на Linux VPS. TLS 1.3 для всего трафика. Корпоративная почта через PGP или S/MIME. Telegram / Signal — только для разрешённых коммуникаций.
RBAC — принцип минимальных прав Каждый сотрудник имеет доступ только к своим ресурсам. Никаких shared паролей. Централизованный IAM (HashiCorp Vault / AWS IAM / Keycloak).
Incident Response Plan Задокументированный план: что делать при взломе, как изолировать машину, кто оповещается, forensics-процедуры. Playbook для разных сценариев.
Security Awareness Training Ежеквартальные учения по социальной инженерии (фишинг-тесты). Новые сотрудники проходят security onboarding до получения доступов.
