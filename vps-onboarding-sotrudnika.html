Как посадить сотрудника
на VPS

Дать удалённому сотруднику рабочую машину в облаке — с изолированной именной учёткой, безопасным доступом и полным контролем у владельца. Два трека: дешёвый Linux (боты, скрипты, разработка) и Windows (1С, офис, специфичный софт). Цены проверены на 23 июля 2026.

Кратко

Рекомендуемый стек под задачу

Выбор зависит от того, какой софт нужен сотруднику и как вам удобно платить. Общий принцип обоих треков: сотрудник получает отдельную непривилегированную учётку (не root/админ), а единственное, что торчит в сеть, — это VPN Tailscale. Порты 22 / 3389 / 5900 наружу не открываются вообще.

Трек Linux — дёшево

Боты, парсеры, скрипты, разработка. Терминал по SSH-ключам + VS Code Remote-SSH; графика (по желанию) — XFCE + xrdp или RustDesk.

  • Оплата из РФ: Aeza — 4 vCPU / 4 ГБ / 60 ГБ NVMe ≈ 1 698 ₽/мес (МИР и крипта)
  • Лучшая цена в мире: Hetzner CX33 — 4 vCPU / 8 ГБ / 80 ГБ NVMe за €8.49 (нужна иностранная карта/PayPal)
  • Дешевле некуда: Contabo VPS 10 — 3 vCPU / 8 ГБ за €4.50
  • ОС: Ubuntu 24.04 LTS + при необходимости лёгкий десктоп XFCE

Трек Windows — 1С, офис

Специфичный Windows-софт: 1С, MS Office, отраслевые программы. Доступ по встроенному RDP (mstsc / «Windows App» на Mac и iPad) с обязательным NLA.

  • Лицензия ВКЛЮЧЕНА в цену: VDSina и RuVDS — главная экономия
  • Осторожно: у Timeweb та же лицензия — доплата ≈ 2 900 ₽/мес
  • Память: под 1С + браузер + офис берите 8 ГБ, не 4
  • За рубежом с легальной лицензией: Contabo Windows (+€4.99) или Amazon Lightsail (лицензия включена)
Слой безопасности для обоих треков — Tailscale (бесплатный mesh-VPN на WireGuard). Ставится на сервер и на устройство сотрудника, работает за NAT без белого IP. Доступ к машине — только внутри приватной сети, а отзыв доступа при увольнении = удаление устройства из панели в один клик.

От заказа до передачи

Пошаговый план

Семь шагов: от выбора сервера до контроля и бэкапов. Команды — для Ubuntu; ключевые Windows-действия отмечены отдельно. Плейсхолдеры вроде ВАШ_IP и worker замените на свои.

1

Выбрать и заказать VPS

Определите трек (Linux или Windows) — от него зависит выбор провайдера и цена лицензии. Память: Linux-GUI — минимум 4 ГБ, Windows-RDP — реально 8 ГБ; vCPU — 2 минимум, 4 комфортно. Провайдера выбирайте под способ оплаты. Для Windows берите провайдера, где лицензия уже включена (VDSina, RuVDS, Lightsail), иначе доплата ≈ 2 900 ₽/мес.

чек-лист заказа
# 1. Трек: Linux (боты/скрипты/разработка) или Windows (1С/офис/спец-софт)
# 2. RAM: Linux — от 4 ГБ, Windows — 8 ГБ; vCPU — 2–4
# 3. Провайдер под оплату: МИР -> Aeza/VDSina; крипта -> Aeza/Vultr; иностр.карта/PayPal -> Hetzner/Contabo
# 4. Образ ОС: Ubuntu 24.04 LTS  ЛИБО  Windows Server с ВКЛЮЧЁННОЙ лицензией
# 5. Оплатить и записать: IP-адрес, логин (root / Administrator), пароль
2

Первичная настройка сервера

Подключитесь под администратором и приведите систему в порядок до того, как заводить сотрудника. Обновите пакеты (непропатченные CVE — главный вектор захвата VPS), задайте таймзону и имя хоста, поставьте базовые инструменты безопасности. Сразу смените дефолтный пароль из письма провайдера.

Ubuntu — первое подключение под root
ssh root@ВАШ_IP
apt update && apt upgrade -y
timedatectl set-timezone Europe/Moscow
hostnamectl set-hostname workbox
apt install unattended-upgrades fail2ban ufw -y     # автообновления, антибрут, файрвол
dpkg-reconfigure -plow unattended-upgrades          # включить автоприменение security-патчей
# Windows: подключиться по RDP как Administrator -> запустить Windows Update, сменить дефолтный пароль
3

Создать непривилегированного пользователя

Один сотрудник = одна именная учётка, никогда не root/admin. Так его действия отделены и логируемы, компрометация не даёт полный контроль над сервером, а доступ легко отозвать. Домашнюю папку закройте правами 700, заведите рабочую директорию с правами 750. На Windows — стандартная учётка только в группе Remote Desktop Users.

Linux — создать пользователя worker
sudo adduser worker
getent group sudo          # убедиться, что worker там НЕТ
sudo chmod 700 /home/worker
sudo mkdir -p /srv/work && sudo chown worker:worker /srv/work && sudo chmod 750 /srv/work
Windows (PowerShell / CMD от админа)
net user worker "Стойкий_Пароль!" /add
net localgroup "Remote Desktop Users" worker /add   # НЕ Administrators!
net localgroup Administrators                        # проверить, что worker не тут
4

Настроить безопасный доступ (SSH-ключи / RDP / Tailscale)

Главное правило: порт SSH / RDP / VNC никогда не торчит в интернет — доступ только внутри приватного tailnet. Сотрудник генерирует ключ у себя (приватный остаётся только у него), присылает вам публичный. Затем хардените sshd, ставите Tailscale, а публичный порт закрываете файрволом. Для Windows — включаете NLA и ограничиваете RDP диапазоном Tailscale.

У сотрудника на его машине
ssh-keygen -t ed25519 -C "worker"   # приватный ключ остаётся у него, вам шлёт файл .pub
На сервере: положить ключ и захарденить SSH
sudo -u worker mkdir -p /home/worker/.ssh && sudo -u worker chmod 700 /home/worker/.ssh
echo "ssh-ed25519 AAAA... worker" | sudo tee /home/worker/.ssh/authorized_keys
sudo chmod 600 /home/worker/.ssh/authorized_keys && sudo chown -R worker:worker /home/worker/.ssh
# /etc/ssh/sshd_config.d/10-hardening.conf:
#   PermitRootLogin no
#   PasswordAuthentication no
#   PubkeyAuthentication yes
#   AllowUsers worker
#   MaxAuthTries 3
sudo sshd -t && sudo systemctl restart ssh
Tailscale + файрвол: закрыть публичный порт
curl -fsSL https://tailscale.com/install.sh | sh
sudo tailscale up --ssh
sudo ufw default deny incoming && sudo ufw default allow outgoing
sudo ufw allow in on tailscale0 to any port 22 proto tcp   # SSH только через VPN
sudo ufw enable            # публичный OpenSSH НЕ открываем
# fail2ban [sshd]: enabled=true, maxretry=4, bantime=1h
sudo systemctl enable --now fail2ban
# Windows: Set-NetFirewallRule -DisplayGroup "Remote Desktop" -RemoteAddress 100.64.0.0/10  + включить NLA + поставить Tailscale
5

Поставить рабочее окружение

Linux без графики — только терминал. Нужен GUI — поставьте лёгкий XFCE и xrdp (подключение любым RDP-клиентом) либо RustDesk. Для разработчика вообще ничего открывать не нужно — хватит VS Code Remote-SSH поверх обычного SSH. Порт xrdp 3389 наружу не светим — только через Tailscale. На Windows десктоп уже есть: под Administrator ставите 1С, MS Office и нужные программы.

Linux — GUI по желанию + инструменты разработчика
# графический десктоп (доступ ТОЛЬКО через Tailscale, порт 3389 публично не открывать):
sudo apt install xfce4 xfce4-goodies xrdp -y
sudo systemctl enable --now xrdp
# для разработчика достаточно VS Code Remote-SSH — наружу открывать нечего:
sudo apt install git build-essential python3-venv nodejs npm -y
# Windows: под Administrator установить 1С / MS Office / нужные программы, ярлыки в профиле worker
6

Передать доступ сотруднику

Отправьте сотруднику приглашение в tailnet (в панели Tailscale → Invite), он ставит Tailscale и логинится — только после этого сервер станет ему виден. Передайте Tailscale-IP машины, имя пользователя и короткую инструкцию. Никогда не давайте пароль root/Administrator — сотрудник работает только под своей учёткой.

Что делает сотрудник
# 1. Принять приглашение в tailnet, установить Tailscale, залогиниться
# 2. Linux-терминал:
ssh worker@100.x.x.x
# 3. Linux GUI / Windows: RDP-клиент (mstsc или "Windows App" на Mac/iPad) -> 100.x.x.x, логин worker
#    либо RustDesk по ID
# Root / Administrator сотруднику НЕ передаём
7

Контроль и бэкапы

Включите автоснапшоты/бэкапы в панели хостера заранее, а не после сбоя (у Beget бесплатные, у Timeweb — платная опция). Настройте оповещения о каждом SSH-входе в Telegram — увидите чужой IP или ночной вход в реальном времени. auditd даёт журнал команд, дисковые квоты не дадут забить диск.

Аудит, квоты, проверки
# включить автоснапшоты/бэкапы в панели хостера (сразу, не после сбоя)
# оповещение о входе: /etc/pam.d/sshd -> session optional pam_exec.so /usr/local/bin/ssh-notify.sh (curl в Telegram)
sudo apt install auditd audispd-plugins -y
# правило /etc/audit/rules.d/work.rules:  -w /srv/work -p rwa -k work_access
sudo augenrules --load && sudo aureport -x --summary
sudo setquota -u worker 5242880 6291456 0 0 /      # дисковая квота ~5 ГБ
# регулярно:
sudo fail2ban-client status sshd ; sudo ausearch -k work_access ; tailscale status

Инструменты доступа — все бесплатные

Ставка на бесплатные и self-hosted инструменты выгодна вдвойне: это и дешевле, и обходит платёжный барьер (у AnyDesk / TeamViewer / Parsec оплата картой из РФ проблемна).

RustDesk

★ 10 · бесплатно

Опенсорс-аналог AnyDesk/TeamViewer. Свой ID+relay на VPS, сквозное шифрование, кроссплатформа. Не блокируется по региону — топ для РФ.

SSH (ключи)

★ 10 · бесплатно

Золотой стандарт доступа к Linux. Ключи ed25519 вместо паролей, фундамент для туннелей и всех остальных инструментов.

Tailscale

★ 9 · free-план

Mesh-VPN на WireGuard. Закрывает порты 22/3389/5900 от интернета полностью, работает за NAT. Обязательный слой безопасности.

RDP / Windows App

★ 9 · бесплатно

Встроенный доступ к Windows VPS. Клиент mstsc на Windows, бесплатное «Windows App» на Mac / iPad. Обязателен NLA + VPN.

xrdp + XFCE

★ 8 · бесплатно

Графический десктоп Linux по RDP. Лёгкий XFCE быстр даже на слабом VPS. Порт 3389 — только через Tailscale.

VS Code Remote-SSH

★ 9 · бесплатно

Для разработчика: правишь код на VPS как локально, поверх обычного SSH. Наружу открывать нечего.

Увольнение

Отзыв доступа за минуты

Продумывайте офбординг с самого начала: именной ключ + Tailscale дают отзыв в один клик. Всё, что сотрудник мог видеть, считается скомпрометированным.

  • Отозвать SSH-ключ: удалить строку из /home/worker/.ssh/authorized_keys — мгновенный отзыв терминального доступа.
  • Удалить из Tailscale: панель → Users → Remove user, деавторизовать все его устройства — доступ пропадает, даже если ключ остался у него.
  • Заблокировать учётку и завершить сессии: sudo usermod -L worker + sudo pkill -u worker; Windows — net user worker /active:no + logoff.
  • Проверить закладки: crontab сотрудника, systemd-user сервисы, лишние ключи, /etc/sudoers.d/worker; просмотреть auditd-логи.
  • Снять снапшот/бэкап рабочих данных до удаления, забрать нужные файлы из /srv/work.
  • Ротация секретов: пароли БД, API-ключи, токены, .env, deploy-ключи, PAT на GitHub/GitLab — всё, что он видел.
  • Панель хостера: если сотрудник знал доступы к личному кабинету — сменить пароль и включить там 2FA.
  • После архивации — удалить учётку: sudo deluser --remove-home worker; Windows — net user worker /delete.
