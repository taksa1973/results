Браузеры — без расширений

Эти браузеры умеют открывать .eth домены прямо из адресной строки — ничего устанавливать не нужно.

🦁
Brave Browser — лучший выбор
Windows macOS Linux Android iOS
Единственный mainstream-браузер с встроенной поддержкой ENS. Просто вводишь site.eth в адресную строку — и сайт открывается. Никаких расширений, никаких шлюзов.
1

Скачай brave.com и установи как обычный браузер

2

Открой настройки: brave://settings/web3 в адресной строке

3

Найди «Resolve Ethereum Name Service (ENS)» → включи если выключено (по умолчанию включено)

4

Вводи site.eth прямо в адресную строку — всё работает

✅ Brave резолвит ENS через собственный узел Ethereum — не через сторонний сервис. Это самый децентрализованный способ из всех.
O
Opera / Opera GX
Windows macOS Android
Opera имеет встроенный Crypto Browser с поддержкой Web3-кошелька и ENS-доменов. Работает без расширений, но требует активации Web3-режима.
1

Скачай opera.com/crypto (Opera Crypto Browser) или обычную Opera

2

При первом запуске настрой Web3 Wallet (встроен в браузер)

3

В настройках → Advanced → Crypto → включи Web3 и ENS

4

Вводи site.eth в адресную строку

⚠️ Opera резолвит ENS через централизованный сервис (не собственный узел). Для максимальной децентрализации предпочти Brave.
Расширения

Chrome, Firefox, Edge — с расширением

Если хочешь остаться в привычном браузере — поставь расширение. Добавляет поддержку .eth в адресную строку.

🦊
MetaMask
Chrome Firefox Edge Brave
Самый популярный Web3-кошелёк. Помимо работы с транзакциями, добавляет поддержку .eth в адресную строку браузера. Когда вводишь site.eth — MetaMask перехватывает и резолвит через ENS.
1

Установи metamask.io/download — выбери свой браузер

2

Создай кошелёк (или импортируй существующий). Seed-фразу запиши на бумаге

3

Убедись что выбрана сеть Ethereum Mainnet в выпадашке сверху

4

Введи site.eth в адресную строку Chrome → появится запрос MetaMask → нажми «Open in MetaMask»

5

MetaMask откроет сайт через встроенный браузер или перенаправит через шлюз

ℹ️ В Chrome при вводе .eth домена может сначала появиться поиск Google — нажми Enter или добавь https:// перед адресом: https://site.eth
IPFS Companion
Chrome Firefox Edge
Расширение от Protocol Labs (создателей IPFS). Позволяет открывать IPFS-пути напрямую: /ipfs/CID... и /ipns/site.eth. Работает через локальный или удалённый шлюз.
1

Установи расширение: github.com/ipfs/ipfs-companion → ссылки на Chrome/Firefox Web Store

2

Опционально: установи IPFS Desktop для запуска локального узла (docs.ipfs.tech)

3

В настройках расширения выбери шлюз: локальный (localhost:8080 через IPFS Desktop) или публичный (cloudflare-ipfs.com)

4

Вводи ipns://site.eth или обычный site.eth — расширение перехватит и откроет через IPFS

💡 IPFS Companion + локальный узел = самый быстрый способ. Файлы кэшируются на твоём компьютере и отдаются мгновенно.

Без установки

HTTP-шлюзы — любой браузер

Если не хочется ничего устанавливать — добавь суффикс к домену. Шлюз переведёт ENS → IPFS → HTML за тебя.

Основной шлюз
https://site.eth.limo
Официальный шлюз от сообщества ENS. Самый актуальный и надёжный. Поддерживает HTTPS, кэширование через Cloudflare.
Альтернатива
https://site.eth.link
Старый шлюз от Cloudflare + ENS. Работает, но eth.limo предпочтительнее — eth.link может быть медленнее.
Cloudflare IPFS
https://cloudflare-ipfs.com/ipns/site.eth
Шлюз от Cloudflare по IPNS-пути. Работает если знаешь CID или ENS-имя. Быстрый за счёт CDN Cloudflare.
IPFS.io
https://ipfs.io/ipns/site.eth
Публичный шлюз от Protocol Labs. Надёжный, но может быть медленнее коммерческих. Хороший запасной вариант.
Dweb.link
https://CID.ipfs.dweb.link
Шлюз по прямому CID. Нужен хэш контента — получаешь в Pinata после загрузки. Минует ENS полностью.
⚠️ HTTP-шлюзы — самый зависимый способ: если шлюз лежит или заблокирован РКН, сайт не откроется. Для надёжного доступа используй Brave или MetaMask.
