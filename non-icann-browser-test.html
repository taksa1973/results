Что установить перед тестом

MetaMask

Кошелёк-расширение. Умеет резолвить ENS (.eth) через встроенный браузер расширения. Для Chrome, Firefox, Edge.

  1. Перейди на metamask.io

  2. Нажми "Download" → выбери браузер

  3. Установи из магазина расширений

  4. Создай или пропусти создание кошелька — для резолвинга он не нужен

Chrome Firefox Edge

UD Browser Extension

Официальное расширение от Unstoppable Domains. Резолвит .crypto, .nft, .x, .wallet, .blockchain, .dao, .888. Не требует настроек.

  1. Открой Chrome Web Store

  2. Поиск: "Unstoppable Domains"

  3. Нажми "Add to Chrome"

  4. Иконка появится в тулбаре — расширение активно

Пошаговый тест по каждому браузеру

🔵

Google Chrome

Нет нативной поддержки. Работает через gateway и расширения.

eth.limo ✓ MetaMask UD Ext
Метод 1 Gateway — без расширений, рекомендовано для UX-теста

  1. Открой Chrome → новая вкладка

  2. В адресную строку введи vitalik.eth.limo → Enter

  3. Что произойдёт: Chrome откроет сайт как обычный HTTPS. Технически ты обратился к домену eth.limo с поддоменом — это легальный ICANN домен, контент идёт из IPFS.

  4. Проверь адресную строку: замочек HTTPS, адрес вида https://vitalik.eth.limo

  5. Повтори для ens.eth.limo и app.uniswap.eth.limo

  6. Зафикси: открылось ли с первого раза? Были ли предупреждения? Скорость vs обычный сайт?

✅ Это прозрачно для пользователя: домен выглядит обычно, HTTPS есть, никаких установок. Пользователь не знает, что это Web3.
Метод 2 MetaMask — попытка открыть .eth напрямую

  1. Убедись, что MetaMask установлен и активен (иконка в тулбаре)

  2. Введи в адресную строку vitalik.eth → Enter

  3. Результат: Chrome покажет результаты Google — MetaMask не перехватывает адресную строку Chrome. Это ожидаемо.

  4. Теперь кликни на иконку MetaMask → найди "Browser" или открой любой dApp сайт → там можно попробовать ENS-резолвинг

  5. Зафикси: MetaMask в Chrome работает для подключения к dApps, но не резолвит .eth в адресной строке самостоятельно.

⚠️ MetaMask не перехватывает URL Chrome. Ввод vitalik.eth ведёт в поиск Google — для обычного пользователя это барьер.
Метод 3 UD Extension — резолвинг .crypto

  1. Убедись, что расширение Unstoppable Domains установлено

  2. Введи в адресную строку unstoppable.crypto → Enter

  3. Расширение перехватит запрос и откроет IPFS контент домена

  4. Что посмотреть: что показывает адресная строка после резолвинга — оригинальный .crypto адрес или IPFS хэш?

  5. Повтори для brad.crypto

🦊

Mozilla Firefox

Экспериментальная ENS-поддержка через about:config + gateway + расширения.

eth.limo ✓ ENS (флаг) UD Ext
Метод 1 Gateway — работает без настроек

  1. Открой Firefox → введи vitalik.eth.limo

  2. Сайт откроется через HTTPS — всё стандартно

  3. Проверь: HTTPS замочек, нет предупреждений, контент загрузился полностью

  4. Сравни скорость загрузки с Chrome (IPFS контент может грузиться медленнее)

Метод 2 Экспериментальный ENS через about:config

  1. Введи в адресную строку about:config → нажми "Принять риск и продолжить"

  2. В строку поиска введи network.trr.mode — убедись, что DoH включён (значение 2 или 3)

  3. Поищи настройку network.ens.enabled — если есть, убедись что значение true

  4. Закрой about:config → введи vitalik.eth в адресной строке

  5. Если настройки нет: Firefox этой версии не поддерживает ENS напрямую. Результат — поиск Google или ошибка.

  6. Зафикси: что случилось — открылось, ошибка, или поиск?

ℹ️ Mozilla экспериментировала с ENS через Cloudflare DNS-over-HTTPS — это не децентрализованно. Firefox просил Cloudflare резолвить ENS, что вызвало споры о приватности. Эксперимент заморожен.
Метод 3 UD Extension для Firefox

  1. Открой addons.mozilla.org

  2. Поищи "Unstoppable Domains" → установи

  3. Введи unstoppable.crypto → проверь резолвинг

  4. Сравни UX с тем же расширением в Chrome

Apple Safari

Минимальная поддержка. Только gateway. Apple не открывает расширениям доступ к URL.

eth.limo ✓ ENS ✗ UD ✗
Метод 1 Gateway — единственный рабочий вариант

  1. Открой Safari → введи vitalik.eth.limo

  2. Сайт откроется нормально — Safari не знает, что это Web3

  3. Попробуй также uniswap.eth.limo — сравни скорость

  4. Проверь: нет ли предупреждения "Safari can't open the page" или "This site is not secure"

Ограничение Почему Web3 расширения не работают в Safari

Apple строго ограничивает расширения Safari через App Store. MetaMask для Safari есть в Mac App Store — но ENS-резолвинг в адресной строке не работает: Apple запрещает расширениям перехватывать URL-запросы.

На iOS Safari вообще нет поддержки Web3 extension API. Для UD расширений нет ни в Mac App Store, ни в iOS App Store.

⚠️ Для пользователей Safari единственный прозрачный путь — eth.limo gateway. Нативного Web3 нет и в ближайшее время не появится из-за политики Apple.
🔷

Microsoft Edge

Chromium-based. Поддерживает те же расширения, что Chrome. Встроенный кошелёк — только для транзакций.

eth.limo ✓ MetaMask UD Ext
Метод 1 Gateway

  1. Открой Edge → введи vitalik.eth.limo

  2. Сайт откроется. Edge может показать SmartScreen предупреждение о "незнакомом сайте" — нажми "Продолжить"

  3. Зафикси: появилось ли предупреждение SmartScreen? Это важно для UX-сравнения с Chrome.

Метод 2 Встроенный Edge Wallet — проверка Web3 настроек

  1. Открой edge://settings/privacy

  2. Найди раздел "Microsoft Edge Wallet" или "Web3"

  3. Проверь, есть ли там что-то про ENS или доменные имена Web3

  4. Введи vitalik.eth в адресной строке — результат будет поиск Bing или ошибка (ENS не резолвится)

  5. Зафикси: встроенный кошелёк Edge существует только для транзакций, а не для резолвинга доменов.

Метод 3 Расширения из Chrome Web Store

  1. Открой edge://extensions/

  2. Включи переключатель "Разрешить расширения из других магазинов"

  3. Открой Chrome Web Store → установи UD Extension

  4. Тестируй так же, как в Chrome — результаты должны быть идентичными

🔴

Opera

Единственный из топ-5 с нативной поддержкой .crypto — партнёрство с Unstoppable Domains.

eth.limo ✓ .crypto нативно ✓
Метод 1 ★ Главный тест Нативный .crypto без расширений

  1. Открой Opera → новая вкладка

  2. Введи в адресную строку: unstoppable.crypto — никаких расширений, никакого .limo

  3. Opera должна автоматически распознать Unstoppable Domain и открыть сайт

  4. Смотри на адресную строку: что там? Протокол? Замочек? Оригинальный .crypto адрес?

  5. Повтори: brad.crypto, reseller.x

  6. Если не открылось: перейди в opera://settings → в поиске введи "Web3" → включи Web3 домены или Crypto Wallets

🎉 Opera — официальный партнёр UD. Поддержка встроена в браузер. Пользователь просто вводит .crypto домен и попадает на сайт — это настоящий нативный опыт без головной боли.
Метод 2 Настройки Web3 в Opera

  1. Открой opera://settings

  2. Перейди в Advanced → Features → Crypto Wallet

  3. Убедись, что Web3 и поддержка доменов включены

  4. В левой боковой панели Opera найди значок кошелька — там можно управлять Web3 настройками

🦁

Brave — Бонус ★ Лучшая нативная поддержка Web3

Нативный ENS (.eth) + встроенный IPFS-узел. Единственный по-настоящему децентрализованный браузер.

ENS нативно ✓ IPFS нативно ✓
Шаг 1 Включить ENS и IPFS в настройках

  1. Открой Brave → введи в адресную строку brave://settings/web3

  2. Раздел IPFS: выбери "Enable IPFS" → "Local node" — это запустит полноценный IPFS-узел прямо внутри браузера

  3. Раздел ENS: убедись, что включено "Resolve ENS domain names"

  4. В выпадающем меню ENS выбери "Ethereum Mainnet" через Brave Wallet (встроенный)

  5. Перезапуск не нужен — настройки применяются сразу

Шаг 2 ★ Главный тест Открыть .eth нативно — без gateway, без расширений

  1. В адресную строку введи: vitalik.eth — просто так, без .limo

  2. Brave обратится к Ethereum блокчейну, найдёт IPFS-хэш, скачает контент через свой локальный IPFS-узел

  3. Brave покажет уведомление: "This site uses ENS (Ethereum Name Service)" → нажми "Proceed"

  4. Смотри на адресную строку: скорее всего увидишь ipfs://Qm... или сохранится vitalik.eth — зафикси

  5. Сайт загрузится. Это настоящий decentralized web — данные из блокчейна и IPFS без централизованного шлюза

  6. Повтори: uniswap.eth, ens.eth

🌐 Brave — единственный способ открыть .eth домен полностью децентрализованно: без доверия к шлюзу eth.limo, без Cloudflare, прямо из блокчейна.
Шаг 3 Бонус: проверить IPFS напрямую

  1. Введи: ipfs://bafybeiemxf5abjwjbikoz4mc3a3dla6ual3jsgpdr4cjr3oz3evfyavhwq/wiki/

  2. Brave откроет страницу из IPFS сети через локальный узел — без серверов, P2P

  3. Зафикси скорость: первая загрузка может быть медленной (поиск пиров), повторная — быстрее

Чек-лист для каждого теста

Что фиксировать в каждом тесте

  • Открылся ли сайт? — да / нет / ошибка
  • Что в адресной строке? — .eth.limo / ipfs:// / .eth / ошибка / поиск
  • Есть ли HTTPS замочек? — безопасное соединение
  • Были ли предупреждения? — SmartScreen, "сайт небезопасен" и т.п.
  • Скорость загрузки — быстро / медленно / таймаут
  • Нужны ли расширения / настройки? — сколько шагов до открытия
  • Понял бы обычный пользователь, что это Web3? — да / нет

Проблемы и решения

❌ Opera не открывает .crypto
opera://settings → Advanced → Features → Crypto Wallet → включи. Или установи расширение UD.
❌ Brave: ошибка ENS резолвинга
brave://settings/web3 → ENS → выбери "Ethereum Mainnet". Подожди 30 сек синхронизации.
❌ eth.limo не открывается
Домен может не иметь IPFS контента. Попробуй vitalik.eth.limo, ens.eth.limo — они точно рабочие.
❌ UD Extension не резолвит
Домен может иметь только адрес кошелька, без IPFS контента. Попробуй unstoppable.crypto.
❌ Safari не открывает .eth или .crypto
Это ожидаемо. Safari не поддерживает Web3 перехват URL. Только eth.limo gateway.

Ожидаемые выводы

✅ Что реально работает для пользователей

  • → eth.limo gateway — любой браузер, HTTPS, нулевой порог входа. Пользователь не замечает Web3.
  • → Opera + .crypto — нативно, просто вводишь домен. Расширения не нужны.
  • → Brave + .eth — полный децентрализованный опыт без шлюзов.
  • → UD Extension в Chromium-браузерах — работает, но требует установки.

❌ Что не работает для обычного пользователя

  • → Прямой набор .eth в Chrome/Firefox/Edge/Safari — ведёт в поиск
  • → Safari + Web3 домены — практически недоступно без gateway
  • → MetaMask в адресной строке — MetaMask не перехватывает URL
  • → .crypto/.x/.nft в Chrome без расширений — не резолвится

💡 Главный вывод

Если хочешь, чтобы сайт открывался у любого пользователя без головной боли — ENS (.eth) + IPFS + eth.limo gateway. Пользователь видит обычный HTTPS адрес, замочек есть, расширений устанавливать не нужно. Технически это Web3 поверх блокчейна — но пользователь этого не замечает.

Для максимальной децентрализации без компромиссов — Brave. Для .crypto нативно и красиво — Opera. Safari — проблемный браузер для Web3 во всех смыслах, только gateway.
