Как это работает в целом

ENS — это имя (домен) на блокчейне. IPFS — это распределённое хранилище файлов. Вместе они дают сайт без единого сервера.

ENS-домен

Имя вида site.eth — запись в смарт-контракте Ethereum. Содержит IPFS-хэш твоего сайта. Покупается раз в год.

IPFS-хранилище

Файлы сайта хранятся по CID — хэшу содержимого. Любой узел, у которого есть твои файлы, может их отдать.

eth.limo

Что такое eth.limo

eth.limo — это HTTP-шлюз. Он переводит ENS-домены в обычные веб-адреса. Не регистратор, не провайдер, не ISP.

🧑‍💻
Пользователь открывает site.eth.limo Обычный HTTPS-запрос, любой браузер без расширений
🔍
eth.limo читает ENS-запись на Ethereum Находит поле contenthash: ipfs://Qm... или /ipfs/baf...
📡
Загружает файлы из IPFS по CID Обращается к пинерам (Pinata, Web3.storage и т.д.), получает данные
✅
Отдаёт HTML/CSS/JS пользователю Как обычный сервер, только содержимое пришло из IPFS
ℹ️ eth.limo — сервис от сообщества ENS. Это не регистратор, не ISP и не хостинг. Просто HTTP-шлюз между обычным вебом и ENS+IPFS.

✓ Что умеет eth.limo

  • Резолвить ENS → IPFS → HTTP
  • Работает в любом браузере без MetaMask
  • Поддерживает contenthash и EIP-3668
  • HTTPS-сертификат (через Cloudflare)
  • Открытый исходный код

✗ Что eth.limo не делает

  • Не регистрирует домены
  • Не хранит твои файлы
  • Не провайдер интернета
  • Не защищает от РКН (сам шлюз может быть заблокирован)
  • Не ускоряет IPFS-загрузку

Пошаговый гайд

Как запустить сайт на ENS

От нуля до работающего сайта. Для тех, кто ни разу не работал с Web3.

1
Установи MetaMask
Скачай расширение MetaMask для Chrome/Firefox. Это твой крипто-кошелёк. Создай новый кошелёк, запиши seed-фразу из 12 слов на бумаге — это единственный способ восстановить доступ.
🚨 Seed-фраза — это твои деньги. Никому не давай, не фотографируй, не сохраняй в облако.
2
Купи ETH
Нужно примерно $25–35 в ETH. Купи на любой бирже (Binance, Bybit, OKX) и выведи на адрес MetaMask.
⚠️ При выводе выбирай сеть Ethereum (ERC-20) — не BSC, не Polygon, именно Ethereum mainnet.
3
Купи ENS-домен
Зайди на app.ens.domains, подключи MetaMask, найди свободное имя (например mysite.eth).
Что Стоимость Тип
Домен 5+ символов / год~$5/годразово + газ
Домен 4 символа / год~$160/годдорого
Домен 3 символа / год~$640/годочень дорого
Газ на регистрацию$5–15gas fee
Газ на обновление сайта$2–8каждый деплой
💡 Регистрируй сразу на 3–5 лет — меньше транзакций с газом.
4
Собери сайт для IPFS
Для React/Vite — настрой HashRouter (не BrowserRouter), иначе страницы кроме главной не откроются. Все пути к ассетам должны быть относительными (./assets/, не /assets/).
// vite.config.js export default { base: './', // ← обязательно для IPFS } // App.jsx — HashRouter вместо BrowserRouter import { HashRouter } from 'react-router-dom'
5
Загрузи файлы на Pinata
Зайди на pinata.cloud, зарегистрируйся, нажми Upload → Folder, выбери папку dist/. После загрузки Pinata даст тебе CID — хэш вида bafybeif...
💡 Pinata даёт 1 GB бесплатно. Для большинства сайтов хватает. Позже добавь ещё пинеров для резервирования.
6
Пропиши contenthash в ENS
В app.ens.domains найди свой домен → Edit Records → Content Hash. Вставь CID в формате:
ipfs://bafybeif...твой_CID_от_Pinata
Подтверди транзакцию в MetaMask. Это будет стоить газ ($2–8).
ℹ️ После записи изменение видно через 1–10 минут (время финализации блока Ethereum).
7
Проверь сайт
Открой твойдомен.eth.limo в браузере. Если всё ок — сайт работает без MetaMask, через обычный HTTPS.
С MetaMask (браузер)
твойдомен.eth — через Brave или MetaMask Mobile
Без MetaMask (все браузеры)
твойдомен.eth.limo — работает везде
Отказоустойчивость

Несколько серверов — сайт не умрёт

IPFS работает как BitTorrent: один и тот же CID может быть запиннен на множестве серверов. Если один упадёт — другие подхватят.

✅ Главное правило: пока хотя бы один пинер хранит твои файлы с нужным CID — сайт доступен. ENS-запись не меняется, домен тот же.

Как пиннить на несколько сервисов

У каждого пинера есть API. После деплоя на Pinata — продублируй тот же CID на другие сервисы:
# Pinata (основной, получаем CID) curl -X POST https://api.pinata.cloud/pinning/pinByHash \ -H "Authorization: Bearer YOUR_JWT" \ -d '{"hashToPin":"bafybeif...ваш_CID"}' # Web3.storage (дублируем) w3 put dist/ --name "mysite" # Или просто — ipfs pin remote add через любой пинер
💡 CID зависит только от содержимого файлов, а не от того, где они лежат. Один и тот же CID на трёх пинерах = три копии одного сайта.

Своё IPFS-ядро на VPS

Полная независимость — никто не может "распиннить" твои файлы.
## Установка на Ubuntu/Debian wget https://dist.ipfs.tech/kubo/v0.27.0/kubo_v0.27.0_linux-amd64.tar.gz tar -xvzf kubo_v0.27.0_linux-amd64.tar.gz && cd kubo && ./install.sh ## Инициализация и запуск ipfs init ipfs daemon & ## Пиннируем папку сайта ipfs add -r dist/ --cid-version=1 # → получаем CID для ENS ## Запуск как сервис sudo systemctl enable ipfs && sudo systemctl start ipfs

Gas Fees

Когда нужен газ, а когда нет

Главное заблуждение: "IPFS = всё стоит газ". На самом деле газ нужен только для операций с Ethereum-блокчейном.

Газ НЕ нужен

Запросы с сайта к бэкенду (REST API, WebSocket)
Запись в базу данных (PostgreSQL, MongoDB)
Авторизация пользователей
Загрузка файлов, изображений
Платежи через Stripe/иной процессинг
Пополнение баланса в казино/игре
Пиннинг файлов на Pinata

Газ нужен

Обновление contenthash в ENS (новая версия сайта)
Регистрация / продление ENS-домена
Установка resolver и других ENS-записей
Смарт-контракты: если твой бэкенд — контракт на Ethereum
Токены, NFT, Web3-транзакции

FAQ

Частые вопросы

Всё что обычно непонятно — тут.

Может ли РКН заблокировать мой сайт на eth.limo? ▾
Да, РКН может заблокировать eth.limo как шлюз — это обычный домен. Но сам сайт в IPFS при этом не исчезнет. Пользователи смогут открыть его через другой шлюз (cloudflare-ipfs.com, ipfs.io) или Brave Browser с встроенным IPFS. ENS-домен в блокчейне — вечен, РКН его не удалит.
Если контент постоянно обновляется — платить газ каждый раз? ▾
Только если меняется фронтенд. Если твой сайт отображает данные с бэкенда в реальном времени (чат, биржа, казино) — бэкенд обновляется без газа. Газ нужен только когда меняются сами файлы React/HTML/CSS, и нужно прописать новый CID в ENS.
IPFS небезопасно — файлы может скачать кто угодно? ▾
IPFS — публичная сеть, как веб. Если ты размещаешь публичный сайт — это нормально. Для приватных данных (ключи, пароли, персональные данные пользователей) — их на IPFS класть не надо. Они хранятся в базе данных на твоём бэкенде, как обычно.
Что значит "несколько пинеров"? Сайт дублируется? ▾
Да. Один и тот же CID (хэш) хранится на Pinata, Web3.storage и твоём VPS одновременно. Если Pinata ляжет — eth.limo получит файлы от Web3.storage. Адрес сайта не меняется — CID один и тот же. Это как BitTorrent: много сидеров = файлы не пропадут.
Можно тянуть данные с обычного сервера с IPFS-фронтенда? ▾
Да. Фронтенд на IPFS — это просто статичный HTML/JS. Он может делать fetch('https://api.myserver.com/...') к любому обычному бэкенду. Нужно настроить CORS на сервере, чтобы разрешить запросы с домена eth.limo.
eth.limo — это провайдер доменов? ▾
Нет. eth.limo — это HTTP-прокси/шлюз. Он не регистрирует домены, не хранит файлы, не является провайдером интернета. Это сервис-переходник: берёт ENS-имя, находит IPFS-файлы, отдаёт в браузер. Как Cloudflare Workers, только для ENS+IPFS.
Как убрать strelnikov23028 из URL? ▾
GitHub Pages всегда включает имя аккаунта в URL (username.github.io). Убрать его можно только через кастомный домен: купи домен ($2–10/год), добавь CNAME-запись в DNS, укажи его в Settings → Pages → Custom domain. После этого URL станет твойдомен.com/game/ вместо username.github.io/results/game/.
