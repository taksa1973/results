ℹ️ IPFS — это не сервер. Это P2P сеть: файлы хранятся у множества участников, каждый файл адресуется по своему хэшу (CID), а не по URL. ENS — это как DNS, только в Ethereum: имя mysite.eth записано в смарт-контракте и указывает на CID.
IPFS

Что такое IPFS

Inter­Planetary File System — P2P сеть хранения данных. Вместо «файл лежит на сервере example.com» — «файл с хэшем Qm… раздаётся тысячами узлов».

Хэш (CID) — это содержимое файла. Если файл изменился — хэш изменился. Это значит: содержимое нельзя подменить незаметно. Пользователь всегда получает именно то, на что ссылается ENS запись.

Что туда загружать

IPFS хранит статические файлы. Ему всё равно, как они сделаны. React после npm run build — это HTML + JS + CSS + картинки. Обычная статика.

Загружаешь папку dist/ → получаешь один корневой CID → ENS запись указывает на него. Пользователь открывает сайт.

ENS

Что такое ENS

Ethereum Name Service — смарт-контракт в блокчейне Ethereum. Работает как DNS, но децентрализованно: никто не может забрать твой домен, заблокировать или изменить без твоего приватного ключа.

Покупаешь mysite.eth на ens.domains → ставишь в запись contenthash значение ipfs://Qm… → всё, домен указывает на IPFS сайт.

Как купить ENS домен

  1. Открой app.ens.domains

  2. Подключи MetaMask кошелёк

  3. Найди свободное имя → зарегистрируй (~$5–20/год + газ)

  4. В настройках записей установи Content Hash → вставь ipfs://ВАШ_CID

  5. Сохрани — транзакция в блокчейне (~$1–5 газа)

  6. Готово: ваш-домен.eth.limo открывается в любом браузере

Три нюанса

⚡ Нюанс 1 — Роутинг

Обычный React Router с HTML5 history (/about, /contact) сломается на IPFS — сеть не знает, как обслуживать эти пути.

// ❌ Сломается на IPFS import { BrowserRouter } from 'react-router-dom' // ✅ Работает import { HashRouter } from 'react-router-dom' // адреса: /#/about, /#/contact
В vite.config.js добавь base: "./" чтобы ассеты грузились по относительным путям.

💎 Нюанс 2 — Каждый деплой = газ

Обновил сайт → изменился IPFS хэш (CID) → нужно обновить ENS запись → транзакция в Ethereum → платишь газ.

Примерные затраты:
Обновление ENS записи$1–5
При нормальном газе (~20 gwei)~$1
При перегрузке сетидо $20
Решение: Fleek автоматизирует деплой и обновление ENS. Ты пушишь в GitHub — они всё делают сами, ты только платишь газ.

🐢 Нюанс 3 — Скорость первой загрузки

IPFS ищет пиров у которых есть твой контент. Если сайт малоизвестный — первый запрос может занять 2–4 секунды.

Деплой

Способ 1 — Fleek (рекомендую)

Коннектишь GitHub → Fleek сам собирает, загружает на IPFS, обновляет ENS. CI/CD для Web3.

  1. Зарегистрируйся на fleek.co

  2. Подключи GitHub репозиторий

  3. Укажи: Framework → React, build command → npm run build, publish dir → dist

  4. В настройках добавь ENS домен → подпиши транзакцию в MetaMask

  5. Теперь каждый git push → автодеплой → обновление ENS

Fleek бесплатный на старте. Платишь только газ за ENS транзакции.

Способ 2 — Вручную (полный контроль)

Сам собираешь, сам загружаешь через CLI или web3.storage / nft.storage.

# 1. Собери React проект npm run build # 2. Установи IPFS CLI npm install -g ipfs ipfs init && ipfs daemon # 3. Загрузи папку dist/ ipfs add -r ./dist # → получишь CID: QmXyz... # 4. Запини (чтобы не удалилось) ipfs pin add QmXyz... # 5. Обнови ENS запись # app.ens.domains → Records → # Content Hash → ipfs://QmXyz...
