macOS на VPS —
правда и мифы

Можно ли запустить macOS на обычном VPS, как к нему подключиться и где реально взять Mac в аренду

Обычный VPS (Hetzner, Contabo, OVH) — нет

Apple EULA запрещает запускать macOS на не-Apple железе. Технически это тоже невозможно: x86/ARM серверы провайдеров не содержат Apple T2/M1 чипов, необходимых для загрузки системы. Хакинтош на VPS — нестабилен и нарушает лицензию.

Аренда Mac в облаке — да, это работает

Специализированные провайдеры устанавливают настоящие Mac mini в стойки дата-центров и сдают их в аренду. Apple официально разрешает это. AWS, MacStadium, Scaleway — все используют физическое Apple-железо.

Почему обычный VPS не вариант
1
Лицензионный запрет Apple macOS EULA §2А прямо запрещает виртуализацию на не-Apple железе. Нарушение = юридические риски для компании.
2
Нет Secure Boot / Apple T-chip macOS 12+ требует Apple T2 или M1/M2 чип для верификации загрузки. На x86 серверах его нет — система не стартует.
3
Хакинтош нестабилен Виртуальный хакинтош (QEMU/KVM + OpenCore) существует, но падает при обновлениях, не поддерживает iCloud, Xcode, подписи приложений.
4
RDP к macOS — не нативно macOS не является RDP-сервером. Нужен VNC (Screen Sharing) или NoMachine. RDP возможен только через сторонние утилиты.

Как подключиться к Mac в облаке
Рекомендуется
Screen Sharing (VNC)
Встроен в macOS. System Settings → General → Sharing → Screen Sharing. Подключение через любой VNC-клиент (RealVNC, TigerVNC, Finder: ⌘K → vnc://IP).
Встроен · Бесплатно
Лучший по качеству
NoMachine
Протокол NX — лучшая скорость и отзывчивость среди всех вариантов. Ставится за 5 минут на Mac. Клиент на Windows/Linux/Mac/iOS. Бесплатен для личного использования.
nomachine.com · Бесплатно
AnyDesk
Простая установка, низкая задержка, хорошо работает на медленном интернете. Хорош для нерегулярного доступа. Бесплатен для личного использования.
anydesk.com · Бесплатно
SSH (терминал)
Только командная строка, но максимальная скорость и безопасность. Включить: System Settings → Sharing → Remote Login. Нужен для настройки VNC/NoMachine.
Встроен · Бесплатно
Microsoft Remote Desktop
Клиент Microsoft может подключаться к Mac, если включён Apple Remote Desktop или поставлен сторонний RDP-сервер. Работает нестабильно. Не рекомендуем.
Сложная настройка
Стандартный RDP
macOS не является RDP-сервером. Подключиться стандартным Windows RDP клиентом к macOS напрямую — невозможно без дополнительных инструментов.
Не поддерживается

Настройка Ubuntu + xRDP за 10 минут
1
Создать VPS (Hetzner CX32)
Ubuntu 22.04, добавить SSH-ключ. Подождать 30 секунд до готовности.
2
Установить XFCE + xRDP
Один скрипт — ставит DE, xRDP-сервер, открывает порт 3389.
# Запустить от root apt update && apt upgrade -y apt install -y xfce4 xfce4-goodies xrdp systemctl enable xrdp # Указать DE для сессии echo startxfce4 > ~/.xsession systemctl restart xrdp
3
Создать пользователя
Не работать под root — создать отдельного пользователя для RDP-сессии.
adduser worker usermod -aG sudo worker # Дать пользователю сессию xfce echo startxfce4 > /home/worker/.xsession chown worker:worker /home/worker/.xsession
4
Подключиться через RDP
Windows: mstsc → ввести IP:3389, логин/пароль пользователя. macOS: Microsoft Remote Desktop из App Store. Linux: Remmina.
# Хост: 123.456.789.0:3389 # Логин: worker # Пароль: ваш_пароль
5
Безопасность (обязательно)
Закрыть порт 3389 от внешнего доступа, открыть только через VPN (Tailscale).
# Tailscale — закрытая сеть curl -fsSL https://tailscale.com/install.sh | sh tailscale up # UFW — только локальная сеть ufw allow from 100.64.0.0/10 to any port 3389 ufw enable
